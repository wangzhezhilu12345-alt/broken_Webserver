# 性能测试

## 环境

Ubuntu：22.04

## 机器配置

CPU：四核

内存:4G

操作系统：Ubuntu22.04

## 压力测试工具

webbench

## 结果及其分析

每秒500个请求的时候

500 clients, running 60 sec.

Speed=1314 pages/min, 181697 bytes/sec.
Requests: 1314 susceed, 0 failed.

能全部收到，每分钟处理了1330个页面。全部返回。

----

每秒5000个请求的时候

5000 clients, running 60 sec.

Speed=1266 pages/min, 177810 bytes/sec.
Requests: 1266 susceed, 0 failed.

每分钟处理的页面比上面少了，但是也能全部成功返回，感觉是服务器到达了负载。

我修改了epoll的最大的事件数量，提升不大，说明目前的写法处理I/O达到的瓶颈。

----

每秒50000个请求的时候

50000 clients, running 60 sec.
problems forking worker no. 8562
fork failed.: Resource temporarily unavailable

直接就是fork的资源不够了。

----

三个过程cpu并没有出现满功率运转的情况。因为服务器测试cpu并不是主要对象。
